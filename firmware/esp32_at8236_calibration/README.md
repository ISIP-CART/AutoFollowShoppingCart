# ESP32 + AT8236 四轮 USB 标定固件

此草图仅用于车轮悬空的四轮电机、编码器和 AT8236 遥测标定。它不启动 WiFi 或 BLE，
不应作为真实遥控固件烧录后落地运行。

## 接线与烧录

- Arduino IDE：`ESP32 Dev Module`，串口 `115200`。
- `GPIO17 -> AT8236 RX2`，`GPIO16 <- AT8236 TX2`，两端共地。
- 烧录后先打开串口监视器；确认车轮完全悬空、附近无人接触车轮。

## USB 命令

- `!U,<interval_ms>`：按 `20..1000 ms` 输出原始 `$MSPD/$MTEP/$MAll` 帧；`!U,0` 关闭。
- `!A,RAISED`：确认车轮悬空后解锁 60 秒。必须先收到有效遥测帧。
- `!T,<seq>,<motor>,<speed>,<duration_ms>`：测试单个电机。
  - `seq` 必须递增；`motor` 为 `1..4`；`speed` 为非零 `[-40,40]`；时长 `100..1500 ms`。
  - 每次只运行一轮，到时自动停车，随后冷却 1000 ms。
- `!S,<seq>`：立即四轮停车并解除解锁。
- `!Q`：查询解锁、遥测、测试、冷却和序号状态。

任何格式错误、遥测超时、解锁超时或行溢出都会立即停车并解除解锁。

## 建议标定顺序

1. 发送 `!U,50`，保存至少 5 秒静止原始帧。
2. 发送 `!A,RAISED`。
3. 对 M1–M4 分别按正、负方向测试 `3、5、8、12、14、20`，每次 300–800 ms；观察是否可靠起转。
4. 每轮执行一次正转后立即停车、冷却、反转，保存原始帧和事件时间。
5. 交换可疑电机与 AT8236 通道后重复，判断问题是否随电机移动。

将完整串口日志连同电机/通道对应关系提供给开发者。只有确定 `$MSPD` 的字段顺序、符号、单位和刷新周期后，正式 BLE 固件才能安全升级为基于实际速度的自动过零换向。
