# ESP32 + AT8236 四轮 USB 标定固件

此固件只用于车轮悬空时标定四个电机和读取 AT8236 遥测，不启动 WiFi 或 BLE，不得烧录后直接落地运行。

## 接线与烧录

- Arduino IDE 选择 `ESP32 Dev Module`，串口波特率 `115200`。
- ESP32 `GPIO17 -> AT8236 RX2`，`GPIO16 <- AT8236 TX2`，两端共地。
- 烧录后确认四轮完全悬空，并准备随时断开电机电源。

## USB 命令

- `!U,<interval_ms>`：按 `20..1000 ms` 采集 `$MSPD/$MTEP/$MAll` 遥测。监视器每 500 ms 合并输出一行最近值，避免刷屏；`!U,0` 关闭采集和显示。
- `!A,RAISED`：确认车轮悬空后解锁 180 秒（3 分钟）。必须先收到有效遥测帧。
- `!T,<seq>,<motor>,<speed>,<duration_ms>`：单轮测试。`motor=1..4`，`speed=[-40,40]` 且不能为 0，时长 `100..1500 ms`；每次结束后冷却 1000 ms。
- `!S,<seq>`：立即四轮停车并解除解锁。
- `!Q`：查询解锁、遥测、测试、冷却和序号状态。

`!U,50` 后示例输出：

```text
CAL,ms=...,event=telemetry,mspd=$MSPD:0.00,0.00,0.00,0.00#,mtep=$MTEP:0,0,0,0#,mall=$MAll:-1407,560,22,1#
```

这是最近帧的汇总，不是每一帧的逐条回显；因此测试命令的 `!OK,T`、`test_start`、`test_stop` 更容易观察。需要停止刷屏时发送 `!U,0`。

## 标定顺序

1. 发送 `!U,50`，观察每 500 ms 一行遥测汇总，静止记录至少 5 秒。
2. 确认收到有效遥测后发送 `!A,RAISED`，预期 `!OK,A`，并看到 `window_ms=180000`。
3. 对 M1–M4 分别执行正负方向阶跃，例如 `!T,1,1,5,1000`、`!T,2,1,-5,1000`；等待每次冷却结束后再发送下一条。
4. 记录起转时间、稳态 `$MSPD`、停止后回零时间、方向符号和异常噪声。
5. 解锁窗口、遥测中断、格式错误或 `!S` 都会立即停车并解除解锁；重新测试前必须再次发送 `!A,RAISED`。
