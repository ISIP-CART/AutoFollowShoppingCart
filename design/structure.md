# 1. 项目基本定义

## 1.1 一句话项目定义

> 本项目旨在：
> “基于 OpenBot 开源平台，构建一台面向室内超市场景、可低速稳定跟随指定用户的低成本自主跟随购物车原型。”

---

## 1.2 项目背景

### 现实背景

传统购物车需要购物者持续手动推动，购物过程中占用双手，在拿取商品、照看随身物品或携带儿童时不够便利；购物结束后，购物车的归位和管理也需要额外人工维护。现有商用跟随机器人或智能购物车成本较高，难以直接作为教学实践和低成本原型验证平台。

### 技术背景

OpenBot 提供了以 Android 手机为核心计算平台的低成本机器人架构，可复用手机摄像头、算力、通信与交互能力；结合轻量级人物检测、目标跟踪、端侧 AI 推理和微控制器底盘控制，可以较低成本实现自主跟随类应用原型。


---

## 1.3 问题定义

目前存在的问题：

1. 传统购物车需要人工推动
2. 传统购物车在购物结束后需要人工维护把它放回指定位置
3. 跟随机器人价格高
4. 低成本平台缺少面向购物场景的应用验证

本项目拟解决的问题：

> 在低成本硬件条件下，实现一台能够在室内平坦环境中低速跟随用户、解决传统购物车人工推车问题的购物车原型。

进一步说，本项目要解决的是一个“可落地的课程原型”问题，而不是直接做商业化产品。系统应优先满足以下要求：

1. 基于 OpenBot 原有架构搭建，避免另起一套高复杂度系统。
2. 在室内平坦环境中实现稳定、可演示的单人跟随闭环。
3. 在目标丢失、急停、通信异常、前方障碍等情况下优先保证安全。
4. 在课程周期和预算内完成从硬件到软件的集成验证。

# 2. 应用场景分析

## 2.1 目标应用场景

| 项目     | 内容 |
| ------ | -- |
| 使用地点   | 室内超市或模拟超市实验场地 |
| 地面条件   | 平坦地面，可存在瓷砖缝、缓坡、转角 |
| 目标用户   | 单个购物者 |
| 小车速度   | 低速，接近步行慢速 |
| 是否承重   | 需要承载购物筐和一定数量商品 |
| 是否有障碍物 | 有，主要为货架、转角、临时摆放物、行人 |
| 是否多人干扰 | 有，可能存在非目标行人穿行 |

---

## 2.2 典型使用流程

1. 用户打开系统 / 启动 OpenBot App。
2. 用户通过手动遥控模式确认底盘、转向、刹停和通信正常。
3. 用户选择“开始跟随”模式，站在小车前方完成目标初始化。
4. 系统检测并锁定目标用户，进入自主跟随状态。
5. 用户在货架区域直行、转弯、短暂停留，小车保持安全距离跟随。
6. 若前方出现障碍物、目标丢失或通信异常，系统减速或停车。
7. 用户主动停止、切换回手动模式或触发急停，系统结束本次运行。

---

## 2.3 演示场景设计

| 演示环节    | 说明 | 成功标准 |
| ------- | -- | ---- |
| 启动系统    | 上电、连接手机与下位机、进入待命状态 | 设备成功启动，手机端能看到控制界面，底盘待命稳定 |
| 手动遥控测试  | 验证前进、后退、转向、停止 | 指令响应正确，无明显失控或延迟过大 |
| 开始跟随    | 用户站到初始化区域，系统锁定目标 | 3 秒内进入跟随状态，目标框或状态提示稳定 |
| 用户直线行走  | 用户以正常慢速向前行走 | 小车连续跟随，不明显蛇形摆动，不撞人 |
| 用户转弯    | 用户经过转角或弯道 | 小车能完成跟随转向且不明显偏离 |
| 用户停下取物  | 用户短暂停留 3-5 秒 | 小车同步减速并停止在安全距离内 |
| 前方出现障碍  | 人为放置纸箱 / 椅子等障碍物 | 小车减速或停车，不与障碍物接触 |
| 目标丢失    | 目标暂时离开视野或被遮挡 | 小车立即取消前进并进入短时原地搜寻；超时后发送 STOP 并停车 |
| 急停 / 停止 | 手动触发急停或停止按钮 | 小车立即停车，解除前不恢复运动 |

# 4. 需求分析

## 4.1 功能需求

| 编号    | 功能需求 | 说明 | 优先级                   | 验收方式 |
| ----- | ---- | -- | --------------------- | ---- |
| FR-01 | 手动遥控 | 用户可通过 OpenBot App 控制底盘前进、后退、转向、停止 | Must | 手动模式下完成基本运动测试 |
| FR-02 | 人物检测与目标锁定 | 系统能够检测视野中的人，并锁定指定用户作为跟随目标 | Must | 初始化后稳定识别目标并进入跟随状态 |
| FR-03 | 自主跟随 | 小车依据目标相对位置输出速度和转向指令，低速跟随用户 | Must | 完成直线、转弯、停下三类场景测试 |
| FR-04 | 模块化购物筐 / 载物平台集成 | 小车具备稳定承载购物筐与轻量商品的结构，首版优先采用“机械限位 + 磁吸辅助固定 + 快拆”的模块化购物筐方案 | Must | 装载后可正常行驶和停止，无明显松动、偏移、倾覆；购物筐可拆装维护 |
| FR-05 | 目标丢失安全搜寻与异常停车 | 目标丢失时系统立即取消前进，进入短时原地搜寻；若超时未找回目标，或出现急停、通信异常、程序异常，则必须停车 | Must | 丢失目标与异常测试中，小车按预期执行搜寻和停车 |
| FR-06 | 障碍物减速 / 停车 | 前方存在障碍物时，小车能够减速或停车避免碰撞 | Should | 障碍测试中不与障碍物接触 |
| FR-07 | 跟随参数可调 | 可调整目标距离、最大速度、转向灵敏度等关键参数 | Should | 至少可通过配置或界面修改并生效 |
| FR-08 | 状态显示与模式切换 | 系统可显示当前模式、连接状态、跟随状态，并支持模式切换 | Should | 用户能完成手动 / 跟随 / 停止切换 |
| FR-09 | 结构快拆与维护 | 购物筐模块、手机支架和电源固定应便于调试、运输和替换，且拆装后能恢复定位 | Should | 拆装后重复完成空载和 1-3 kg 负载低速行驶测试 |

## 4.3 性能指标

| 指标       | 目标值 | 测试方法 | 是否必须达成 |
| -------- | --- | ---- | ------ |
| 跟随速度     | 0.3-0.8 m/s | 直线跟随测试，测量一定距离内用时 | 是 |
| 跟随距离     | 1.0-2.0 m 可调 | 在不同参数下测量目标与车体间距 | 是 |
| 连续跟随距离   | 不少于 30 m | 在室内路线中连续行走测试 | 是 |

| 负载能力     | 首版按原型机口径实现购物筐 + 1-3 kg 物品稳定运行；更高载重以后续底盘实测为准 | 负载测试 | 是 |
| 最小转弯半径   | 不大于 1.0 m | 标记路线转弯测试 | 否 |
| 障碍停车距离   | 距障碍物至少保留 0.3 m 安全余量 | 障碍接近测试 | 是 |
| 单次运行时间   | 不少于 20 min | 满电条件下连续演示测试 | 是 |

## 5.2 详细用例

### UC-01 手动遥控小车

| 项目   | 内容         |
| ---- | ---------- |
| 参与者  | 用户 |
| 前置条件 | 小车已上电，手机与下位机连接正常 |
| 基本流程 | 1. 用户打开 OpenBot App；2. 进入手动模式；3. 执行前进、后退、转向、停止操作；4. 观察底盘响应 |
| 异常流程 | 若通信异常或控制异常，系统停车并提示切换到安全状态 |
| 后置条件 | 完成底盘联调，系统可进入跟随模式 |
| 成功标准 | 基本运动方向正确，停止可靠，无明显失控或严重延迟 |

---

### UC-02 开始自主跟随

| 项目   | 内容         |
| ---- | ---------- |
| 参与者  | 用户、系统 |
| 前置条件 | 手动模式测试通过，摄像头与检测模型工作正常 |
| 基本流程 | 1. 用户站到初始化区域；2. 在 App 中切换到跟随模式；3. 系统检测并锁定目标用户；4. 小车开始低速跟随 |
| 异常流程 | 若未检测到用户或目标不稳定，系统保持待命，不输出运动指令 |
| 后置条件 | 系统进入 FOLLOW 状态 |
| 成功标准 | 能稳定锁定目标并完成起步跟随，不出现明显误跟随 |

---

### UC-03 目标丢失后安全搜寻与停车

| 项目   | 内容         |
| ---- | ---------- |
| 参与者  | 系统 |
| 前置条件 | 小车已处于 FOLLOW 状态 |
| 基本流程 | 1. 目标因遮挡或离开视野而丢失；2. 系统立即将线速度置零，取消继续前进的跟随指令；3. 系统进入 LOST 状态并启动短时搜索计时；4. 小车仅允许原地低速左右扫描，不允许继续前向位移；5. 若在阈值内重新检测到目标，则回到 FOLLOW；6. 若超时仍未找回目标，则发送 STOP 指令并进入 STOP 状态 |
| 异常流程 | 若搜索期间检测到障碍、急停或通信异常，则立即进入 OBSTACLE、EMERGENCY 或 STOP 状态 |
| 后置条件 | 小车停车，等待重新初始化目标或人工干预 |
| 成功标准 | 目标丢失后小车不继续盲目前进，短时搜寻动作受限且超时后可靠停车 |

---

### UC-04 前方障碍停车

| 项目   | 内容         |
| ---- | ---------- |
| 参与者  | 系统 |
| 前置条件 | 小车处于 FOLLOW 或 MANUAL 慢速行驶状态，障碍检测链路有效 |

| 异常流程 | 若障碍检测不稳定，系统应优先采取保守停车策略 |
| 后置条件 | 系统进入 OBSTACLE 或 STOP 状态 |
| 成功标准 | 小车不与障碍物接触，停车位置可控 |


## 7.2 数据流

| 数据          | 来源 | 去向 | 作用 |
| ----------- | -- | -- | -- |
| 图像帧         | 手机摄像头 | 手机端感知与决策模块 | 获取人物和环境信息 |
| 人物检测结果      | 感知模块 | 决策模块 | 判断目标是否存在及其位置 |
| 目标位置 / 距离估计 | 决策模块 | 跟随控制器 | 生成速度与转向控制量 |
| 速度指令        | 手机端控制模块 | 微控制器 | 下发线速度 / 转向速度目标 |
| 电机控制信号      | 微控制器 | 电机驱动模块 | 驱动左右轮或差速底盘 |
| 障碍物信息       | 障碍传感器 / 视觉模块 | 安全模块 / 决策模块 | 触发减速、停车或绕障策略 |
| 急停信号        | 用户 / 安全模块 | 微控制器 / 电机驱动 | 强制停止运动输出 |
| 状态信息        | 微控制器、手机端 | App 界面 / 日志 | 展示连接状态、模式状态、异常状态 |

## 7.3 控制流程

```text
开始
  ↓
系统初始化
  ↓
手机与下位机建立连接
  ↓
手动模式自检：遥控、转向、停止、通信状态确认
  ↓
选择模式：手动遥控 / 自主跟随
  ↓
若为自主跟随：
  图像采集 → 人物检测 → 目标锁定 → 跟随控制计算 → 发送速度指令
  ↓
下位机接收指令 → 电机执行
  ↓
安全检查：目标丢失 / 障碍物 / 急停 / 通信异常 / 程序异常
  ↓
若目标丢失：
  立即取消前进速度 → 进入 LOST → 原地低速短时搜索 → 找回目标则恢复 FOLLOW，超时则发送 STOP
  ↓
其他异常则停车并切换状态
  ↓
正常则继续运行
```

---

## 7.4 状态机设计

| 状态            | 说明 | 进入条件 | 退出条件 |
| ------------- | -- | ---- | ---- |
| INIT 初始化      | 系统上电、自检、建立连接 | 开机 | 自检完成并进入待命 |
| MANUAL 手动模式   | 用户通过 App 遥控底盘 | 用户选择手动模式 | 用户切换到 FOLLOW 或 STOP |
| FOLLOW 自主跟随   | 系统锁定目标并输出跟随控制 | 完成目标初始化 | 目标丢失、障碍触发、急停、人工停止 |
| LOST 目标丢失     | 系统暂时失去目标，立即取消前进速度并启动安全搜寻计时 | FOLLOW 中目标检测失效 | 进入 SEARCH、重新锁定目标或进入 STOP |
| SEARCH 目标搜索  | 执行短时原地低速左右扫描，不允许继续前向位移 | LOST 后启用搜索策略 | 找回目标回到 FOLLOW，超时则 STOP |
| OBSTACLE 障碍停车 | 前方障碍触发安全停车 | 障碍判断成立 | 障碍解除并经人工确认恢复 |
| EMERGENCY 急停  | 急停或严重故障状态 | 用户急停、通信故障、程序故障 | 人工复位后回到 STOP |
| STOP 停止       | 车辆静止待命 | 人工停止或异常停车结束 | 用户重新选择模式 |

# 17. 今日讨论结论

## 17.1 已确定事项

1. 首版定位为基于 OpenBot 的缩比原型验证，不承诺真实商用购物车级载重。
2. 首版载物目标按“购物筐 + 1-3 kg 物品”进行稳定演示和测试，更高载重只作为后续实测扩展。
3. 结构方案优先采用模块化购物筐，当前实现思路为“机械限位 + 磁吸辅助固定 + 快拆 + 防滑缓冲”。
4. 磁吸只作为辅助固定和拆装便利手段，主要抗位移能力由机械限位、快拆件、螺丝 / 扎带 / 卡扣等结构件承担。
5. OpenBot Android 开发从 `dev/OpenBot/android` 开始，OpenBot 代码改动先提交到 `dev/OpenBot` 子仓库，再回到主仓库提交 submodule 指针。
