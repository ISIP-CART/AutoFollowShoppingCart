# 03 智能购物车与跟随载物产品竞品调研

## 1. 调研目的

本文用于回答以下问题：

1. 市场上存在的智能购物车、跟随行李箱、载物跟随机器人和服务机器人平台各自的能力边界是什么。
2. 哪些能力可以为本项目的低成本购物车跟随原型提供借鉴。
3. 哪些能力属于商业产品闭环，首版不应追求复制。

本文调研的产品分为四类：

- **商用智能购物车**：以零售场景为目标的智能购物车产品。
- **自动跟随行李箱**：以个人出行载物为目标的跟随产品。
- **载物跟随机器人**：以货物运输和跟随为目标的机器人。
- **服务机器人平台**：通用型移动服务机器人（作为参照坐标）。

## 2. 分类调研

### 2.1 商用智能购物车

#### Amazon Dash Cart

| 维度 | 内容 |
| --- | --- |
| 产品定位 | Whole Foods / Amazon Fresh 门店内的智能购物车 |
| 核心能力 | 计算机视觉 + 重量传感器自动识别放入商品；内置屏幕显示购物清单和价格；自动结账（走出店门自动扣款） |
| 是否自主跟随 | **否**。由用户手动推行，不具备自主移动或跟随能力 |
| 感知系统 | 多摄像头阵列（顶部和筐内）+ 重量传感器 |
| 成本 | 非零售产品，不对外销售 |
| 交付状态 | 已在部分 Whole Foods 门店部署 |

**来源说明**：Amazon Dash Cart 的公开部署信息来自已确认的新闻报道和亚马逊官方发布；部分页面因 URL 变更返回 404，但产品能力描述来自多项独立科技媒体报道的交叉验证。

#### Instacart Caper Cart

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 超市智能购物车，Instacart 于 2021 年以约 3.5 亿美元收购 |
| 核心能力 | 计算机视觉识别商品 + 称重；触摸屏交互（优惠推送、店内导航）；自动结算；与 Instacart 平台深度整合 |
| 是否自主跟随 | **否**。由用户手动推行 |
| 感知系统 | 摄像头 + 重量传感器 + 触摸屏 |
| 成本 | 面向零售商 B2B 销售，非个人消费产品 |
| 交付状态 | 已在美国多个连锁超市部署 |

**来源说明**：Instacart 收购 Caper AI 的信息来自多项独立财经与科技媒体报道的交叉验证；具体产品页当前返回 404。

#### 国内智能购物车（超嗨 / 多点等）

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 国内超市的智能购物车改造方案 |
| 核心能力 | 条码扫描 / 商品识别；防损防盗；部分集成自助结算和广告屏 |
| 是否自主跟随 | **否**。仍然由用户手动推行 |
| 感知系统 | 条码扫描器为主，部分加入摄像头做商品识别 |
| 成本 | B2B 方案，面向零售商 |
| 交付状态 | 已在国内部分超市部署 |

**来源说明**：超嗨科技等信息来自公开商业报道和行业展会的交叉验证。因无法直接访问企业页面获取规格表，部分细节为行业普遍特征。

#### 商用智能购物车小结

当前市面上的"智能购物车"产品几乎全部聚焦于**商品识别 + 自动结算**，而不是**自主移动 + 跟随**。它们的核心价值主张是"跳过排队结账"而非"解放双手不用推车"。这意味着本项目要做的跟随能力在现有商用智能购物车中几乎找不到直接参照。

### 2.2 自动跟随行李箱

#### ForwardX Ovis

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 消费级自动跟随智能行李箱 |
| 核心能力 | 基于计算机视觉的侧后方跟随（无需手持信标）；避障；防丢警报；TSA 认证可拆卸电池 |
| 跟随方式 | 视觉跟随（摄像头捕捉用户身形轮廓） |
| 载重能力 | 行李箱自重约 4.5 kg |
| 续航 | 约 20 km |
| 价格 | 约 $799 |
| 交付状态 | 已量产，面向消费者零售 |

**来源说明**：ForwardX Ovis 的产品信息来自多项科技产品评测的交叉验证，为行业知名产品。

#### Airwheel SR5

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 可骑行自动跟随智能行李箱 |
| 核心能力 | 跟随模式 + 可骑行模式（内置电机可载人低速行驶）；UWB 定位跟随 |
| 跟随方式 | UWB / 超声波跟踪 |
| 载重能力 | 可载成人骑行（约 100 kg）+ 行李箱储物 |
| 价格 | 约 $600-800 |
| 交付状态 | 已量产 |

**来源说明**：Airwheel 为知名智能出行品牌，产品信息来自其官网公开页面（已确认官网可访问）和电商平台产品页。

#### Travelmate Robotics

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 全向自动跟随行李箱 |
| 核心能力 | 全向轮底盘；手机 App 遥控 + 跟随；内置 GPS 追踪 |
| 跟随方式 | 手机蓝牙 / App 信标跟随 |
| 载重 | 标准登机箱容量 |

**来源说明**：Travelmate 为众筹平台上的明星项目，信息来自众筹页面和技术媒体报道的交叉验证。

#### 自动跟随行李箱小结

自动跟随行李箱是本项目在**形态上**最接近的参照物——它们都是在室内地面移动的载物设备，并且具备跟随能力。关键差异在于：

- 行李箱是**消费电子产品**（整机设计、电池内置、外观精细），而本项目是**课程原型**。
- 行李箱的跟随方案多样化：纯视觉（Ovis）、UWB（SR5）、蓝牙信标（Travelmate），可以为本项目提供技术路线参考。
- 行李箱的载重通常为个人行李级别（<20 kg），与本项目的购物筐载物需求类似。
- 行李箱产品通常价格较高（$400-800），不适合作为本项目硬件平台直接采购。

### 2.3 载物跟随机器人

#### Piaggio Gita

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 个人货物跟随机器人，由 Piaggio Fast Forward（Vespa 制造商旗下）开发 |
| 核心能力 | 基于计算机视觉的人物跟随；可承载 18 kg 货物；360° 障碍感知；最高速度约 10 km/h |
| 跟随方式 | 视觉跟随（通过摄像头和深度传感器锁定并跟随用户） |
| 外形 | 圆筒形，26 升储物空间，两个大轮子 |
| 价格 | 约 $3,250 |
| 交付状态 | 已量产，面向消费者零售 |

**来源说明**：Piaggio Gita 信息来自其官网 mygita.com 的已确认访问内容；具体技术规格来自多项科技产品评测的交叉验证。

#### Burro (原 Auro)

| 维度 | 内容 |
| --- | --- |
| 产品定位 | 农业跟随运输机器人 |
| 核心能力 | 跟随田间工作人员运输农作物；自主返航；可承载 200+ kg |
| 跟随方式 | 视觉 + 传感器跟随 |
| 载重能力 | 200+ kg |
| 场景 | 户外农田 |
| 价格 | B2B 租赁/销售，数万美元级别 |
| 交付状态 | 已在农场部署 |

#### 载物跟随机器人小结

Gita 是最接近本项目概念的商用产品——它是一台"会跟着人走的载物机器人"。但它价格高昂（$3,250），且是完整商业产品，包含工业设计、电池管理、售后和合规等大量非技术工作。本项目从 Gita 可以获得的核心启发是：

- 跟随距离和跟随策略（侧后方跟随 vs 正后方跟随）。
- 不需要用户手持任何信标的"无感跟随"体验。
- 360° 障碍感知的重要性。

## 4. 借鉴 vs 不复制

### 4.1 借鉴清单

| 借鉴来源 | 借鉴内容 | 如何体现在本项目中 |
| --- | --- | --- |
| **Gita** | 无信标视觉跟随体验；侧后方跟随策略；≤2 m 安全跟随距离 | 用手机摄像头做人物检测跟随，不依赖用户佩戴信标 |
| **Gita** | 360° 障碍感知 | 通过超声波（前方）+ 视觉（手机）实现基础障碍感知 |
| **自动跟随行李箱** | 低速运行（≤步行速度）；目标丢失后停车的安全策略 | 速度上限控制在 0.3-0.8 m/s；目标丢失立即取消前进 |
| **自动跟随行李箱** | 载物平台与重心设计 | 购物筐安装在底盘低重心位置；限位 + 磁吸辅助固定 |
| **Gita / 跟随行李箱** | 跟随形态（用户在前、设备在后） | 跟随距离 1.0-2.0 m，不紧贴用户 |
| **商用智能购物车** | 购物筐形态和容量参考 | 使用标准购物筐，容量约 30-50 L |
| **商用智能购物车** | 场景定位：超市/货架通道/室内平坦地面 | 演示场景设计参照超市场地布局 |
| **OpenBot** | 低成本手机主脑架构；差速底盘控制 | 作为核心技术平台底座 |
| **高成本机器人平台** | 反面提醒：控制范围，首版不做全栈导航 | 限定为单目标低速跟随 + 异常停车 |

### 4.2 不复制清单

| 不复制内容 | 原因 |
| --- | --- |
| **商品识别与自动结账** | 涉及 CV + 重量传感器融合、支付系统对接、数据库建设和长期运维，远超课程范围 |
| **云端多车调度** | 需要云端平台、门店系统对接和稳定网络，首版是单台原型 |
| **完整 SLAM 与路径规划** | 需要多传感器融合、建图和复杂算法，不符合首版"相对跟随"需求 |
| **自动归位与跨区域导航** | 需要全局定位和路径规划，属于完整自主导航范畴 |
| **商业级工业设计** | 需要专业模具、材料、供应链和合规认证，不适合单台课程原型 |
| **门店系统 / 会员系统 / POS 集成** | 纯软件工程和系统集成工作，不涉及核心机器人能力 |
| **复杂人机交互（语音、大屏广告等）** | 非首版核心能力，会分散工程精力 |
| **高成本传感器（激光雷达、深度相机等）** | 不符合预算约束，且对首版跟随任务来说过度 |
| **移动端之外的额外计算平台（树莓派等）** | 与 `OpenBot + 手机主脑` 架构硬约束冲突 |

## 5. 本项目差异化分析

| 维度 | 商用智能购物车 | 跟随行李箱 | 载物跟随机器人 | **本项目** |
| --- | --- | --- | --- | --- |
| 跟随能力 | 无 | 有 (但面向个人出行) | 有 (但价格高) | **有 (低成本 + 购物场景)** |
| 载物能力 | 有 (但需手推) | 有 (但容量有限) | 有 (但价格高) | **有 (购物筐 + 课程级预算)** |
| 安全设计 | 无动力 (天然安全) | 基本安全 | 商业级安全 | **多层安全 (急停 + 超时 + 障碍)** |
| 开源与可复现性 | 闭源 | 闭源 | 闭源 | **全开源** |
| 价格 | 非零售 / 高 | $600-800 | $3,250+ | **1500-3000 元** |
| 场景匹配 | 超市 (但无跟随) | 机场/出行 (非超市) | 通用 (非超市) | **超市/模拟场地 + 跟随** |

**核心差异点**：市场上同时具备"自主跟随"和"室内购物载物"两个能力的产品几乎不存在。商用智能购物车不做跟随，跟随行李箱和 Gita 不做购物场景。本项目的价值在于填补这个交集——用低成本原型验证"购物车自主跟随"这一核心能力的可行性。

## 7. 后续建议

1. 持续关注 Gita 和自动跟随行李箱的用户体验评测，尤其是"跟随稳定性""目标丢失后的行为""障碍物处理"等场景表现。
2. 将竞品的跟随距离（1-2 m）、安全速度上限（步行速度）、障碍物响应方式作为本项目参数调节的参考基线。
3. 不追求在首版中复制任何商业产品的外观、支付或门店系统集成能力。
