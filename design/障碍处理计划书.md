# 自主跟随购物车项目——障碍处理计划书

> 适用模块：上位机 / Android 端 / Human Cart Simulator
> 推荐方向：手机视觉驱动的局部可通行空间估计与跟随式避障
> 核心目标：从“检测前方危险并停车”升级为“判断左 / 中 / 右可通行空间，并生成低速局部避障动作”
> 当前约束：优先只使用手机摄像头与手机端视觉算法，暂不依赖外接 ToF、超声波、激光雷达等硬件。

---

# 1. 计划定位

本计划不再把障碍处理限定为：

```text
视觉风险检测 + 减速 / 停车
```

而是将其升级为：

```text
手机视觉局部可通行空间估计
+
跟随任务下的低速局部避障
```

也就是说，系统不只判断“前方有没有危险”，而要进一步判断：

```text
左侧能不能走？
中间能不能走？
右侧能不能走？
目标用户在哪个方向？
哪一个低速动作最安全？
```

最终输出不应只是：

```text
OBSTACLE_CAUTION
OBSTACLE_STOP
```

而应输出：

```text
FOLLOW_CLEAR
FOLLOW_CAUTION
AVOID_LEFT
AVOID_RIGHT
BLOCKED_WAIT
TARGET_OCCLUDED
EMERGENCY_STOP
```

---

# 2. 核心思想

## 2.1 不以“识别所有障碍物类别”为目标

超市场景中的障碍物可能包括：

```text
货架
路人
购物车
柱子
纸箱
展架
地面杂物
目标用户自己
其他顾客的腿
临时堆放物
```

如果系统试图识别所有类别，会导致模型复杂、漏检风险高、调试困难。

因此，本项目采用更实用的定义：

> **障碍 = 出现在购物车未来运动空间中的非可通行区域或高风险物体。**

系统不必知道它是“纸箱”还是“柱子”，只需要知道：

```text
它是否挡住了购物车接下来要走的区域？
```

---

## 2.2 从“障碍检测”转向“可通行空间估计”

本项目更适合关注：

```text
哪里可以走？
```

而不是：

```text
哪里有障碍？
```

推荐核心任务：

```text
输入：手机相机画面
输出：左 / 中 / 右 三个方向的可通行评分
```

例如：

```text
left_free_score = 0.82
center_free_score = 0.24
right_free_score = 0.67
```

则系统可以判断：

```text
中间被阻挡
左侧更安全
优先尝试低速左绕
```

---

## 2.3 避障不是完整路径规划

本项目暂不追求：

```text
建图
定位
全局路径规划
复杂绕障路径生成
```

而是采用更符合购物车跟随场景的策略：

```text
跟随目标用户
+
评估局部可通行空间
+
选择低速候选动作
```

这可以称为：

> **Leader-aware Local Avoidance：面向跟随目标的局部避障。**

目标用户提供“导航意图”，手机视觉提供“局部风险判断”，控制器选择安全的低速动作。

---

# 3. 阶段目标

本计划直接跳过“只能可靠停车”的阶段，从以下目标开始：

## 3.1 第一目标：左 / 中 / 右可通行空间判断

系统应能将画面前方划分为三个基本方向：

```text
LEFT
CENTER
RIGHT
```

并实时输出：

```text
left_free_score
center_free_score
right_free_score
```

同时输出：

```text
left_risk
center_risk
right_risk
```

以及：

```text
overall_confidence
failure_reason
```

---

## 3.2 第二目标：局部避障动作选择

基于左 / 中 / 右空间评分，系统生成候选动作：

```text
STOP
SLOW_FORWARD
LEFT_ARC
RIGHT_ARC
ROTATE_LEFT
ROTATE_RIGHT
WAIT_AND_REACQUIRE
```

选择原则：

```text
优先安全
其次保持目标方向
再次保证动作平滑
最后考虑绕行效率
```

---

## 3.3 第三目标：跟随目标与避障融合

系统不是独立避障，而是在跟随目标时避障。

因此需要同时考虑：

```text
目标用户方向
目标用户距离状态
左 / 中 / 右可通行空间
非目标行人风险
目标是否被遮挡
目标是否正在绕过障碍
```

目标行为：

```text
如果目标从障碍左侧绕行，且左侧可通行，则购物车尝试低速左绕。

如果目标从障碍右侧绕行，且右侧可通行，则购物车尝试低速右绕。

如果目标被障碍遮挡，且可通行空间不确定，则停车等待，而不是盲目前进。
```

---

# 4. 总体架构

建议将原计划中的：

```text
ObstacleRiskEstimator
```

升级为：

```text
LocalTraversabilityEstimator
+
LocalAvoidancePolicy
```

整体模块如下：

```text
PhoneVisionObstacleSystem
├── TargetRegionManager
├── GroundRegionEstimator
├── FreeSpaceEstimator
├── PersonObstacleAnalyzer
├── RelativeDepthRiskEstimator
├── MotionRiskEstimator
├── LocalRiskMapFusion
├── LocalTraversabilityEstimator
└── LocalAvoidancePolicy
```

---

# 5. 模块职责

## 5.1 TargetRegionManager

作用：

```text
区分目标用户和非目标物体。
```

目标用户不应被简单当作普通障碍物，但仍应参与安全距离判断。

输出：

```text
target_bbox
target_mask
target_center
target_direction
target_distance_state
target_visibility_state
```

注意：

```text
目标用户 = 跟随对象
但目标用户太近时 = 高风险对象
```

---

## 5.2 GroundRegionEstimator

作用：

```text
判断画面下半部分中哪些区域更可能是地面。
```

可用线索：

```text
图像位置
颜色连续性
纹理连续性
边缘变化
透视关系
历史帧稳定区域
```

首版不要求完美语义分割，只需要输出粗略地面概率：

```text
ground_probability_map
```

---

## 5.3 FreeSpaceEstimator

作用：

```text
根据地面区域推断左 / 中 / 右可通行空间。
```

输出：

```text
left_free_score
center_free_score
right_free_score
```

基本逻辑：

```text
画面下方区域中，连续地面越多，可通行分越高。
非地面区域、人体、货架边缘、近处突起越多，可通行分越低。
```

---

## 5.4 PersonObstacleAnalyzer

作用：

```text
分析非目标行人是否进入购物车运动区域。
```

输入：

```text
person detections
target_id
target_bbox
```

输出：

```text
non_target_person_risk
left_person_risk
center_person_risk
right_person_risk
```

规则：

```text
目标用户：用于导航，但不能撞上。

非目标行人：如果进入运动走廊，则视为动态障碍。
```

---

## 5.5 RelativeDepthRiskEstimator

作用：

```text
使用单目深度模型生成相对深度风险图。
```

建议定位：

```text
辅助判断近处风险
辅助比较左 / 中 / 右哪个区域更开阔
不直接输出绝对米制距离
```

输出：

```text
relative_depth_map
near_risk_map
left_depth_clearance
center_depth_clearance
right_depth_clearance
depth_confidence
```

注意：

单目深度应作为增强模块，而不是唯一判断依据。

---

## 5.6 MotionRiskEstimator

作用：

```text
使用光流或帧间变化判断碰撞趋势。
```

它不需要知道真实距离，只关注：

```text
某个区域是否正在快速接近？
```

输出：

```text
left_motion_risk
center_motion_risk
right_motion_risk
time_to_collision_proxy
```

适用场景：

```text
路人突然横穿
目标突然停下
购物车正在靠近货架
画面下方物体快速变大
```

---

## 5.7 LocalRiskMapFusion

作用：

```text
融合地面、深度、人体、运动等信息，生成局部风险地图。
```

输入：

```text
ground_probability_map
free_space_scores
depth_risk_map
person_risk
motion_risk
target_region
```

输出：

```text
left_risk
center_risk
right_risk
front_stop_risk
overall_scene_confidence
```

融合原则：

```text
只要某一方向存在高置信度危险，该方向风险升高。

如果多个模块互相矛盾，则降低整体置信度。

如果整体置信度太低，则进入 BLOCKED_WAIT 或 FOLLOW_CAUTION。
```

---

## 5.8 LocalTraversabilityEstimator

作用：

```text
判断每个方向是否可通行。
```

输出：

```text
LEFT_CLEAR / LEFT_CAUTION / LEFT_BLOCKED
CENTER_CLEAR / CENTER_CAUTION / CENTER_BLOCKED
RIGHT_CLEAR / RIGHT_CAUTION / RIGHT_BLOCKED
```

建议数据结构：

```text
TraversabilityResult
├── left_state
├── center_state
├── right_state
├── left_score
├── center_score
├── right_score
├── confidence
├── dominant_risk_source
└── failure_reason
```

---

## 5.9 LocalAvoidancePolicy

作用：

```text
根据可通行空间和目标方向选择动作。
```

输出：

```text
suggested_action
action_confidence
control_hint
```

候选动作：

```text
SLOW_FORWARD
LEFT_ARC
RIGHT_ARC
ROTATE_LEFT
ROTATE_RIGHT
BLOCKED_WAIT
EMERGENCY_STOP
```

---

# 6. 局部空间划分

## 6.1 基础三分区

首版建议将画面下半部分划分为：

```text
LEFT
CENTER
RIGHT
```

其中：

```text
LEFT：画面左侧 1/3
CENTER：画面中间 1/3
RIGHT：画面右侧 1/3
```

但实际计算时，应重点关注：

```text
画面下半部分
+
购物车未来运动区域
```

而不是整张图平均计算。

---

## 6.2 运动走廊

固定三分区之后，下一步应引入“运动走廊”概念。

不同动作对应不同风险区域：

```text
SLOW_FORWARD：中间走廊
LEFT_ARC：左前方弧形走廊
RIGHT_ARC：右前方弧形走廊
ROTATE_LEFT：左侧近距离区域
ROTATE_RIGHT：右侧近距离区域
```

也就是说，系统不只问：

```text
画面中间有没有障碍？
```

而是问：

```text
如果我执行这个动作，未来 1 秒内会不会撞到东西？
```

---

# 7. 决策逻辑

## 7.1 基础决策规则

```text
if emergency_risk:
    EMERGENCY_STOP

else if target_lost and scene_uncertain:
    BLOCKED_WAIT

else if center_clear and target_center_near_center:
    SLOW_FORWARD

else if center_blocked and left_clear and target_on_left:
    LEFT_ARC

else if center_blocked and right_clear and target_on_right:
    RIGHT_ARC

else if center_blocked and left_clear and right_blocked:
    LEFT_ARC

else if center_blocked and right_clear and left_blocked:
    RIGHT_ARC

else if all_blocked:
    BLOCKED_WAIT

else:
    FOLLOW_CAUTION
```

---

## 7.2 目标用户绕障逻辑

如果目标用户从障碍物一侧绕行：

```text
目标 bbox 位于左侧
center_blocked
left_clear
```

则：

```text
AVOID_LEFT
```

如果：

```text
目标 bbox 位于右侧
center_blocked
right_clear
```

则：

```text
AVOID_RIGHT
```

如果目标不可见：

```text
target_lost
```

则不根据最后一次位置盲目绕行，应进入：

```text
BLOCKED_WAIT
```

或短时：

```text
WAIT_AND_REACQUIRE
```

---

## 7.3 非目标行人处理

如果非目标行人进入中间走廊：

```text
center_person_risk high
```

则：

```text
center_blocked
```

如果左侧或右侧仍可通行：

```text
left_clear or right_clear
```

则可低速绕行。

如果行人正在横穿，且光流或连续帧显示其快速进入车前区域：

```text
motion_risk high
```

则优先：

```text
BLOCKED_WAIT
```

不主动抢行。

---

# 8. 状态机设计

建议新增或升级以下状态：

```text
FOLLOW_CLEAR
FOLLOW_CAUTION
AVOID_LEFT
AVOID_RIGHT
BLOCKED_WAIT
TARGET_OCCLUDED
TARGET_LOST
EMERGENCY_STOP
```

状态说明：

| 状态                | 含义                  | 运动策略           |
| ----------------- | ------------------- | -------------- |
| `FOLLOW_CLEAR`    | 目标可见，中心通路清晰         | 正常低速跟随         |
| `FOLLOW_CAUTION`  | 可通行但置信度下降           | 降速，保持观察        |
| `AVOID_LEFT`      | 中间受阻，左侧可通行          | 低速左弧线          |
| `AVOID_RIGHT`     | 中间受阻，右侧可通行          | 低速右弧线          |
| `BLOCKED_WAIT`    | 左 / 中 / 右均不可通行或信息矛盾 | 停止等待           |
| `TARGET_OCCLUDED` | 目标被障碍或路人遮挡          | 停止或低速原地调整视角    |
| `TARGET_LOST`     | 目标丢失                | 停止 / 搜索 / 重新确认 |
| `EMERGENCY_STOP`  | 高危碰撞风险              | 立即停车           |

注意：

停车仍然存在，但不再是唯一目标，而是候选动作之一。

# 9. Control 输出策略

## 9.1 动作到 Control 的映射

```text
SLOW_FORWARD:
    Control(+v, +v)

LEFT_ARC:
    Control(v_left_low, v_right_high)

RIGHT_ARC:
    Control(v_left_high, v_right_low)

ROTATE_LEFT:
    Control(-turn_low, +turn_low)

ROTATE_RIGHT:
    Control(+turn_low, -turn_low)

BLOCKED_WAIT:
    Control(0, 0)

EMERGENCY_STOP:
    Control(0, 0)
```

---

## 9.2 速度原则

所有避障动作必须低速。

```text
FOLLOW_CLEAR:
    low_speed

FOLLOW_CAUTION:
    very_low_speed

AVOID_LEFT / AVOID_RIGHT:
    very_low_speed

BLOCKED_WAIT:
    zero

EMERGENCY_STOP:
    zero
```

不允许：

```text
高速绕障
高速追赶
目标丢失后继续前进
左右不确定时强行选择一侧
```

# 11. 开发阶段安排

## 阶段 A：三方向可通行评分

目标：

```text
实现 LEFT / CENTER / RIGHT 三方向 free score。
```

输入：

```text
camera frame
person detections
target bbox
```

输出：

```text
left_free_score
center_free_score
right_free_score
```

验收：

```text
能在 Human Cart Simulator 中实时显示左 / 中 / 右可通行评分。
```

---

## 阶段 B：非目标行人风险融合

目标：

```text
将目标用户与非目标行人区分开。
```

实现：

```text
目标用户不作为普通障碍处理。
非目标行人进入运动走廊时，提高对应方向风险。
```

验收：

```text
路人从左 / 中 / 右经过时，系统能正确提升对应区域风险。
```

---

## 阶段 C：局部风险地图

目标：

```text
从三方向评分升级为局部风险地图。
```

输出：

```text
left_risk
center_risk
right_risk
front_stop_risk
```

验收：

```text
系统能判断中间受阻但左侧或右侧仍可通行。
```

---

## 阶段 D：候选动作选择

目标：

```text
基于可通行空间选择避障动作。
```

实现动作：

```text
SLOW_FORWARD
LEFT_ARC
RIGHT_ARC
BLOCKED_WAIT
```

验收：

```text
当前方中间被阻挡时，系统能根据左右空间建议左绕或右绕。
```

---

## 阶段 E：单目深度辅助

目标：

```text
引入单目深度作为风险热力图。
```

原则：

```text
深度只辅助判断，不单独决定动作。
```

验收：

```text
当深度图显示某方向近处风险升高时，对应方向 risk 上升。
```

---

## 阶段 F：光流 / TTC 辅助

目标：

```text
判断动态接近风险。
```

适用：

```text
路人横穿
目标突然停下
购物车快速靠近障碍
```

验收：

```text
运动风险升高时，系统能进入 FOLLOW_CAUTION 或 BLOCKED_WAIT。
```

# 12. 验收场景

## 12.1 静态障碍

| 场景        | 预期行为           |
| --------- | -------------- |
| 中间放纸箱，左侧空 | `AVOID_LEFT`   |
| 中间放纸箱，右侧空 | `AVOID_RIGHT`  |
| 左中右都被挡    | `BLOCKED_WAIT` |
| 中间空，左右有障碍 | `SLOW_FORWARD` |

---

## 12.2 动态行人

| 场景      | 预期行为                              |
| ------- | --------------------------------- |
| 路人从左侧经过 | 左侧风险升高                            |
| 路人从中间横穿 | `BLOCKED_WAIT` 或 `FOLLOW_CAUTION` |
| 路人从右侧经过 | 右侧风险升高                            |
| 路人与目标交叉 | 不切换目标，降低速度或等待                     |

---

## 12.3 目标用户绕障

| 场景         | 预期行为                               |
| ---------- | ---------------------------------- |
| 目标从左侧绕过障碍  | 若左侧可通行，则 `AVOID_LEFT`              |
| 目标从右侧绕过障碍  | 若右侧可通行，则 `AVOID_RIGHT`             |
| 目标被障碍遮挡    | `TARGET_OCCLUDED` / `BLOCKED_WAIT` |
| 目标消失后无法重识别 | `TARGET_LOST`                      |

---

## 12.4 超市场景

| 场景       | 预期行为                              |
| -------- | --------------------------------- |
| 货架通道直行   | `FOLLOW_CLEAR`                    |
| 通道中间有人停留 | `FOLLOW_CAUTION` 或 `BLOCKED_WAIT` |
| 货架边缘靠近车身 | 对应侧风险升高                           |
| 前方出现购物车  | 中间风险升高，判断左右可通行性                   |
| 目标拐入货架侧边 | 若目标可见且侧向通路可行，则低速跟随转向              |

# 14. 风险与降级策略

## 14.1 可通行空间误判

风险：

```text
地面反光、货架阴影、复杂纹理可能被误判。
```

应对：

```text
引入历史帧稳定性。
引入单目深度辅助。
低置信度时进入 FOLLOW_CAUTION 或 BLOCKED_WAIT。
```

---

## 14.2 目标与障碍混淆

风险：

```text
系统可能把目标用户当成障碍，或者把非目标行人当成目标。
```

应对：

```text
TargetRegionManager 必须与 PersonMatcher 联动。
目标用户参与安全距离判断，但不作为普通障碍处理。
非目标行人进入运动走廊时提高风险。
```

---

## 14.3 左右绕行选择错误

风险：

```text
左侧看似可通行，但实际空间不足。
```

应对：

```text
所有绕行动作必须低速。
绕行过程中持续重新评估 left/right risk。
若风险升高，立即进入 BLOCKED_WAIT。
```

---

## 14.4 单目深度不稳定

风险：

```text
单目深度输出相对深度，不一定等于真实距离。
```

应对：

```text
不直接使用深度值作为唯一控制依据。
只将其作为 risk source 之一。
当深度与地面/检测结果冲突时，降低 confidence。
```

---

# 15. 当前计划改写建议

建议将原计划中的：

```text
ObstacleRiskEstimator
```

改为：

```text
LocalTraversabilityEstimator
+
LocalAvoidancePolicy
```

建议将原计划中的描述：

```text
画面下方 40%
画面中间 50%
如果该区域出现近距离深度异常或非目标前景物体，则进入风险状态。
```

改为：

```text
系统不再只检测固定 ROI 中的前方风险，而是将手机画面下半部分划分为左 / 中 / 右三个候选通行方向，并结合目标用户位置、非目标行人检测、地面可通行区域、单目深度风险图和帧间运动风险，生成局部可通行空间评分。控制层基于该评分在低速直行、左绕、右绕、等待等候选动作中选择最安全动作，从而实现面向跟随任务的局部避障。
```

# 16. 最终总结

本障碍处理计划的核心不是：

```text
看到障碍就停车
```

而是：

```text
判断哪里能走，并在跟随目标的前提下选择安全方向。
```

推荐路线：

```text
左 / 中 / 右可通行空间判断
↓
局部风险地图
↓
目标用户方向融合
↓
候选避障动作评分
↓
低速跟随式局部避障
```

该方案相比原计划的固定 ROI 风险检测有三个提升：

1. 从“前方危险检测”升级为“局部可通行空间估计”。
2. 从“减速 / 停车”升级为“左绕 / 右绕 / 等待 / 跟随”的候选动作选择。
3. 从“普通避障”升级为“面向目标用户跟随的局部避障”。

最终目标：

> 构建一个手机-only 的跟随式局部避障系统：目标用户提供导航意图，手机视觉判断局部可通行空间，购物车在低速条件下选择直行、左绕、右绕或等待。
