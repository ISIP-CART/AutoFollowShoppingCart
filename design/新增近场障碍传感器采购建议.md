# 新增近场障碍传感器采购建议

更新时间：2026-07-10

适用对象：硬件采购、ESP32 下位机、整车集成同学

## 1. 提出原因

PC 单目深度实验发现：按暂定手机安装参数（相机高度 0.65 m、向下俯角 12°），小车前探 20 cm 和原地转向 20°时的新增车体扫掠区位于手机画面下方。手机单目深度可以继续提供中距离空间结构证据，但不能单独承担车体近场硬防撞。

因此建议补充直接连接 ESP32 的近场测距模块，由 ESP32 在 Android、USB 通信或深度模型失效时仍能独立停车。

推荐职责划分：

```text
Android 手机：人物跟随、中距离深度、行为决策
ESP32：近场传感器采集、数据有效性检查、硬停车、安全状态
AT8236：四电机驱动与编码器闭环
物理急停：直接切断电机驱动电源
```

## 2. 推荐采购结论

### 2.1 推荐完整方案

```text
前方中央：URM09 I2C 超声波 ×1
左前角：VL53L1X ToF ×1
右前角：VL53L1X ToF ×1
```

选择理由：

- 中央超声波波束较宽，适合发现正前方墙面、纸箱、人体等大目标；
- 左右 ToF 视场较窄，适合检查货架边缘和原地转向时的两个前角扫掠区；
- 仅使用一个超声波，避免多个超声波同时工作产生串扰；
- 三个传感器都可接入 ESP32 的 3.3 V I2C 总线，不占用 AT8236 的 UART；
- ESP32 可以把三个读数的最小有效值继续通过 OpenBot 原生 `s<distance_cm>` 格式上报 Android。

### 2.2 第一批建议清单

| 优先级 | 采购项 | 明确规格 | 数量 | 预算参考 | 用途 |
| --- | --- | --- | ---: | ---: | --- |

| P0 | VL53L1X ToF 载板 | VL53L1X；带稳压、电平转换；引出 `VIN/GND/SDA/SCL/XSHUT` | 2 | 30–100 元/个 | 左右车角和原地转向保护 |
| P0 | PH2.0-4P / I2C 连接线 | 与 URM09、ESP32 扩展板接口匹配 | 2–3 | 5–20 元 | 传感器接线与备用 |

## 3. 下单时必须核对

### 3.1 URM09

必须确认商品是：

```text
DFRobot Gravity URM09
SKU: SEN0304
通信接口: I2C
```

不要误买以下同名变体：

```text
URM09 Analog 模拟量版
URM09 Trig 单线触发版
其他 UART / RS485 型号
```

需要向卖家确认：

1. 是否包含 PH2.0-4P 连接线；
2. 是否支持 3.3 V 供电与 3.3 V I2C；
3. 模块版本及默认 I2C 地址；
4. 是否提供 Arduino / ESP32 示例代码和寄存器说明。

### 3.2 VL53L1X

必须确认购买的是完整载板而不是裸芯片。载板至少需要：

```text
板载稳压
SDA/SCL 电平转换
XSHUT 引脚
3.3 V 供电可用
```

两个 VL53L1X 上电后的默认 I2C 地址相同。ESP32 需要分别控制两个 `XSHUT`，依次启动并改成不同地址，例如：

```text
左侧 ToF  → 0x2A
右侧 ToF  → 0x2B
```

不要购买：

- 未引出 `XSHUT` 的载板；
- 只标注“VL53”但不明确芯片型号的模块；
- 将 VL53L0X 冒充 VL53L1X 的商品；
- 需要 5 V I2C 高电平且没有电平转换的模块。

## 4. 暂定接线方案

建议建立购物车专用 ESP32 配置，不直接照抄 OpenBot `DIY_ESP32` 的引脚表。原配置中 GPIO5/18 同时用于超声波和指示灯，不适合本项目直接复用。

暂定分配：

| ESP32 资源 | 连接对象 | 说明 |
| --- | --- | --- |
| USB UART0 | Android 手机 | OpenBot USB Serial，保留 GPIO1/3 |
| UART2 GPIO16/17 | AT8236 | 首选电机驱动通信链路，最终以卖家协议为准 |
| GPIO21 SDA | 三个近场传感器 | 3.3 V I2C 总线 |
| GPIO22 SCL | 三个近场传感器 | 3.3 V I2C 总线 |
| GPIO25 | 左 VL53L1X `XSHUT` | 上电地址分配 |
| GPIO26 | 右 VL53L1X `XSHUT` | 上电地址分配 |
| GPIO32 | 急停状态输入 | 物理急停仍需硬线切断电机电源 |

接线约束：

1. ESP32、AT8236 和传感器必须共地；
2. ESP32 扩展板 GVS 电压跳线优先设为 3.3 V；
3. 若 VL53L1X 载板把 SDA/SCL 上拉到 VIN，则 VIN 必须使用 3.3 V，不能用 5 V；
4. 传感器电源线远离电机线，必要时使用绞线并增加去耦电容；
5. AT8236 的 UART 电平仍需向卖家确认，不能默认其 TX 对 ESP32 是 3.3 V 安全电平。

## 5. 安装位置建议

### 5.1 中央超声波

- 安装在车体前方中央，离地约 15–25 cm；
- 基本水平，最多轻微向下 0–5°；
- 探头必须伸出购物筐、底板和横梁形成的遮挡区域；
- 使用减震垫，避免底盘和麦轮振动引起读数波动。

### 5.2 左右 ToF

- 安装在两个前角，离地约 10–20 cm；
- 分别向左前、右前偏转约 20–30°；
- 避免正对车体自身支架、透明亚克力罩或高反光金属件；
- 支架应允许后续调整角度，而不是一次粘死。

安装完成后应在俯视图中确认三个视场覆盖：

```text
左前角扫掠区 + 正前方宽区域 + 右前角扫掠区
```

## 6. ESP32 固件最低要求

建议新增独立 `SafetyRangeManager`，不要把安全逻辑散落在电机控制代码中。

最低功能：

```text
读取 center / left / right 三路距离
维护 valid / invalid / stale 状态
3点或5点中值滤波
独立于 Android 执行硬停车
向 Android 上报最小有效距离
记录触发硬停车的传感器和原因
```

第一版阈值建议：

| 条件 | 建议动作 |
| --- | --- |
| 前进时任一有效距离 ≤30 cm | 立即停止并请求 AT8236 制动 |
| 距离恢复到 >40 cm 且连续 3 帧有效 | 解除距离锁存，但等待新的运动指令 |
| 原地转向时对应车角距离 ≤20 cm | 禁止该方向转向 |
| 任一关键读数超过 200 ms 未更新 | 标记 `STALE` |
| 运动中全部关键传感器均无效或超时 | 保守停车，不得当作“前方空旷” |

OpenBot 原固件在超声波未收到回波时会用最大距离代替。本项目必须改成明确的无效状态，避免断线或传感器故障被误判为安全。

Android 首版接口可继续复用：

```text
功能查询：f...:s:
传感器回传：s<min_valid_distance_cm>\n
采样周期请求：s100\n
```

左/中/右单独数据和错误位可在后续诊断协议中扩展，但不应阻塞 ESP32 本地硬停车。

## 7. 到货验收与测试

### 7.1 单模块验收

每个传感器分别测试：

```text
10 cm / 20 cm / 30 cm / 50 cm / 100 cm 静态距离
连续运行 10 分钟是否丢失或冻结
黑色衣物、纸箱、墙面、人体、货架边缘
断开信号线后是否被识别为 INVALID，而不是最大距离
```

### 7.2 三模块联调

1. I2C 扫描应发现 URM09 和两个不同地址的 VL53L1X；
2. 三路同时采样时连续 10 分钟无总线卡死；
3. 电机悬空运行时距离噪声仍在可接受范围；
4. 前进时中央障碍触发 30 cm 停车；
5. 原地左转、右转时对应前角障碍能禁止动作；
6. 拔掉任意模块后 ESP32 能上报失效状态；
7. 拔掉全部测距模块后运动指令不能继续驱动车辆前进。

### 7.3 暂不作为通过条件

以下内容不应在第一轮硬件验收中要求：

- 用距离传感器识别左通道或右通道；
- 由 ToF / 超声波直接规划转弯路线；
- Android 根据三路距离自动绕障；
- 多传感器替代物理急停按钮。

## 8. 备选与暂缓采购

### 8.1 A02YYUW UART 超声波

A02YYUW 具备防水外壳、3–450 cm 量程和较小盲区，可作为 URM09 的防水替代。但它使用 UART，而当前架构已优先将 UART2 分配给 AT8236；引入额外 UART 会增加引脚和固件复杂度，因此暂不作为首选。

### 8.2 HC-SR04

HC-SR04 成本低且 OpenBot 原生代码容易复用，但常见版本 Echo 为 5 V，需要分压或电平转换，裸露探头和接线的工程可靠性也较低。适合桌面验证，不建议直接作为购物车最终硬停车传感器。

### 8.3 多个超声波

暂不建议购买三个超声波替代“一超声波 + 两 ToF”。多个超声波需要错峰触发并处理串扰，货架走廊中的多径反射也可能增加误报。

## 9. 采购前需要团队确认的问题

请采购和下位机同学共同确认：

1. 是否接受推荐完整方案 `URM09 ×1 + VL53L1X ×2`；
2. VL53L1X 具体商品是否明确包含稳压、电平转换和 `XSHUT`；
3. ESP32 扩展板能否稳定提供 3.3 V I2C 总线；
4. AT8236 最终使用 UART 还是 I2C，默认波特率和逻辑电平是多少；
5. 是否已经采购常闭式物理急停按钮及保险丝；
6. 谁负责设计和打印/加工三个传感器的可调安装支架；
7. 是否先购买 1 个 URM09 完成硬停车最小验证，再下单两个 ToF。
